Executive Summary

SignalFlow brings reactive programming to Unreal Engine 5 with robust core architecture, automatic network replication, and full Blueprint integration.

üéØ The Problem

UI Polling Overhead: UMG property bindings poll every frame regardless of changes, wasting CPU cycles across all platforms.

Replication Complexity: Network replication requires extensive boilerplate code for each replicated property.

Blueprint Communication: Inter-Blueprint data sharing requires complex casting chains and event dispatchers.

GAS Attribute Limitation: Epic's Gameplay Ability System only supports float attributes (FGameplayAttributeData).

Actor Overhead: Each replicated value requires an Actor (~500+ bytes + tick), inefficient for data-heavy games on any platform.

üöÄ SignalFlow Solution

True Reactivity: UI updates only when data actually changes. Zero polling overhead.

Automatic Replication: Add a flag and values sync automatically via FFastArraySerializer.

Global Signal Registry: Any Blueprint can access any signal by name - no casting required.

Type-Agnostic Attributes: FSignalSet stores any type (int, bool, FVector, structs) - not just floats.

Actor-Decoupled Data: Singleton replicates all signals. 16-byte handles, event-driven. Zero per-value Actor overhead.

üí° Why This Matters for All Platforms

Unreal Engine's Actor-centric architecture creates significant overhead for data-heavy games (MMOs, idle games, RPGs with many stats). Each replicated value traditionally requires an Actor with its own replication channel, tick function, and component hierarchy. SignalFlow's singleton replication and event-driven updates eliminate this overhead, improving performance on PC, console, and mobile alike.

Why SignalFlow?

Compare traditional Unreal Engine patterns with SignalFlow's reactive approach.

‚ùå Traditional UE5: Health Bar ~45 lines

// Header file DECLARE_DYNAMIC_MULTICAST_DELEGATE_OneParam( FOnHealthChanged, float, NewHealth); UCLASS() class AMyCharacter : public ACharacter { UPROPERTY(ReplicatedUsing=OnRep_Health) float Health; UPROPERTY(BlueprintAssignable) FOnHealthChanged OnHealthChanged; UFUNCTION() void OnRep_Health(); }; // CPP file void AMyCharacter::OnRep_Health() { OnHealthChanged.Broadcast(Health); }

‚úÖ SignalFlow: Same Result 6 lines

// Create replicated signal auto Health = Signal(100.0f, "Player.Health", ESignalFlags::Replicated); // Widget auto-updates on change HealthBar->BindToSignal(Health); // Change value - UI updates automatically Health.Set(75.0f);
BENEFITS
  • No UPROPERTY/UFUNCTION boilerplate
  • No manual delegate binding
  • Automatic replication with one flag
  • UI updates only when data changes

‚ùå Traditional: Damage Calc Manual

‚úÖ SignalFlow: Automatic Auto

// Define source signals auto BaseDamage = Signal(10.0f); auto WeaponDamage = Signal(25.0f); auto Multiplier = Signal(1.5f); // Computed signal - auto-updates! auto FinalDamage = (BaseDamage + WeaponDamage) * Multiplier; // Change any input ‚Üí auto update WeaponDamage.Set(50.0f);

Advanced Patterns: Lambda & Pipe Operators

üîÆ Computed with Lambda Explicit

Lambda receives values, not signals. Dependencies auto-tracked.
